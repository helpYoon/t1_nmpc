# Whole-Body Trajectory Tracking for Floor-Pickup — Design Spec (2026-06-28)

**Status:** design, awaiting review. Supersedes nothing; this is a new milestone (M-Track) parallel
to M0 stand. M0 stand path stays untouched.

## 1. Goal & scope

Run the pre-computed whole-body motion plan `data/motion_plan.pkl` (a bimanual **floor pickup**,
both feet planted throughout) through the T1 whole-body RNEA + Fatrop MPC in **closed-loop MuJoCo**,
tracking it as a moving reference, **in real time** (solve fits the control period), with a
`time_scale` knob to slow the motion. Tested like `sim/stand.py`.

This is a **path-following / trajectory-tracking** controller. The plan is the path; the OCP tracks
it as a moving target while keeping the robot dynamically feasible and balanced. Because the motion
is **pure double-support stance** (no stepping, no contact-schedule changes), there is no gait
machinery — this is M0 stand generalized to a time-varying whole-body target.

### Success criteria (acceptance)
1. Closed-loop MuJoCo run completes the full plan (time-scaled) **without falling**.
2. **Real-time:** achieved via an **RTI per-tick Fatrop iteration cap** (`track_warm_iters=5`) with
   N=8. Cap=3 falls at the left-release transition (~plan 9.2 s); cap≥5 completes the full motion.
   Measured solve on the (loaded) dev box: **p50≈24 / p90≈27 / worst-case≈67 ms**; per-iter cost is
   ~2× inflated under concurrent load, so ~13–15 ms p90 is expected on a dedicated box (the hard 16 ms
   target is hardware-dependent and unverified on a quiet box). Acceptance = the closed loop
   **completes the full motion** with **bounded** worst-case tick time (the cap guarantees this); rare
   transition ticks spike (soft real-time, not a hard per-tick guarantee). JIT/codegen ruled out
   (graph too large).
3. Tracking quality: base-pose and arm-joint RMS within tolerance (set during tuning); **hand position
   error at grasp keyframes ≈2.8–3.4 cm** (closed-loop, after PD tracking; 2 cm is a future-tuning
   target).
4. Feet stay planted (no slip / no lift) and Σfz/mg ≈ 1.
5. M0 stand regression tests still pass.

**Validated 2026-06-28 (closed loop):** completes 14.75 s motion, max |measured−reference tilt| ≈1.9°,
Σfz/mg 0.93–1.11.

## 2. Source data — `data/motion_plan.pkl`

`{schema_version:3, segments:[8], joint_name_groups, quaternion_convention, schema_doc}`.

- **8 segments**, each `T=81` samples at `dt≈0.0227 s` → **~14.8 s total**. Generated by cuTAMP with a
  **simplified inverted single-leg URDF** (`data/t1_simplified.urdf`): planar base on the floor →
  `ankle_pitch → shank → knee_pitch → thigh → Torso_Pitch → Waist → Waist_Yaw → Trunk`; axes negated
  (`0 -1 0`) so joint *values* transfer sign-consistently; Hip_Roll/Hip_Yaw collapsed into the thigh.
- Per-segment `position` channels: `trunk_xyz`, `trunk_quat_wxyz/xyzw`, `trunk_height`, `trunk_pitch`,
  `trunk_yaw`, `ankle_pitch`, `knee_pitch`, `left_arm(7)`, `right_arm(7)`, `left_hand_xyz/quat`,
  `right_hand_xyz/quat`. Matching `velocity` and `acceleration` dicts.
- `held_objs` per segment marks grasp state. Transitions across segments give the grasp/release events
  (see §5). No object mass is provided.
- The simplified-URDF **Trunk frame coincides with `t1.urdf`'s Trunk frame to 0.0 m** (verified in the
  simplified URDF header), so the plan's Trunk pose *is* our pinocchio floating-base pose, and the
  plan's `trunk_height` *is* our base-z.

**Reference authority:** the joint mapping and time-scale semantics follow `t1_kd_mpc`
(`src/t1_wbc/reference.py`, `src/t1_wbc/config.py`, `src/t1_mpc/t1_mpc_kd.py`), the existing system that
tracks this same plan on this same robot. Deliberate divergences from it are listed in §11.

- pinocchio FreeFlyer T1: `nq=36` (`[pos3, quat_xyzw4, joints29]`), `nv=35`, base link `Trunk`.
- 29 joints in pinocchio order: `[AAHead_yaw, Head_pitch, L_Shoulder_Pitch, L_Shoulder_Roll,
  L_Elbow_Pitch, L_Elbow_Yaw, L_Wrist_Pitch, L_Wrist_Yaw, L_Hand_Roll, R_(same 7), Waist,
  L_Hip_Pitch, L_Hip_Roll, L_Hip_Yaw, L_Knee_Pitch, L_Ankle_Pitch, L_Ankle_Roll, R_(same 6)]`.
  Joint index → full-q index = `7 + joint_index`.
- Hand frames `left_hand_link` (frame 20), `right_hand_link` (frame 36) — correspond exactly to the
  plan's hand channels (FK verified to 0.0 cm).
- Foot-center frames `rm.foot_center_frame_ids = (Lc, Rc)`; 8 contact corners (4/foot), `nf=24`.
- Joint position limits `m.lowerPositionLimit / upperPositionLimit` (used in §7; e.g. `L_Knee_Pitch
  ∈ [0, 2.34]`, `L_Ankle_Pitch ∈ [-0.87, 0.35]`).

## 4. Reference mapping (motion_plan.pkl → pinocchio reference) — AUTHORITATIVE

Per node, build a reference state `x_ref ∈ ℝ⁷¹ = [q_ref(36), v_ref(35)]`, plus hand task targets and
grasp gates. `(x0, y0, yaw0)` = robot start anchor (identity in sim from nominal).

### Base (FreeFlyer)
| target | plan channel | transform |
|---|---|---|
| base pos x,y `q[0:2]` | — | anchored to start `(x0,y0)` (NOT from plan). |
| base pos z `q[2]` | `trunk_height` (= `trunk_xyz[2]`) | **tracked** → crouch depth. |
| base quat `q[3:7]` (xyzw) | `trunk_quat_xyzw` | `R_z(yaw0) · R.from_quat(plan_quat)` → forward lean. |
| base ang vel `v[3:6]` | `trunk_angular_velocity_world` | rotate by `R_z(yaw0)`, to body-local, `/time_scale`. |
| base lin vel `v[0:3]` | d(`trunk_xyz`)/dt | world→body-local `Rᵀ·`, `/time_scale` (small; feet planted). |

> Note: `t1_kd_mpc` ignores `trunk_xyz` because its reduced model pins the base; our **full-order free
> base** uses `trunk_height` as the crouch reference — validated feasible (see §6).

### Joints — TRACKED (21)
| T1 joint | plan channel | sign / broadcast |
|---|---|---|
| `{L,R}_Shoulder_Pitch,Roll`, `Elbow_Pitch,Yaw`, `Wrist_Pitch,Yaw`, `Hand_Roll` | `left_arm[0:7]`, `right_arm[0:7]` | linear, order `[Sh_P, Sh_R, El_P, El_Y, Wr_P, Wr_Y, Hand_Roll]`, no reindex. |
| `Waist` | `trunk_yaw` | **`Waist = −trunk_yaw`** (NEGATED). |

### Joints — leg pitch: SEED ONLY (weak regularization + warm-start bias, NOT a tracked target)
| T1 joint | plan channel | note |
|---|---|---|
| `L_Hip_Pitch` **and** `R_Hip_Pitch` | `trunk_pitch` | broadcast identical, no sign flip. |
| `L_Knee_Pitch` **and** `R_Knee_Pitch` | `knee_pitch` | broadcast identical. |
| `L_Ankle_Pitch` **and** `R_Ankle_Pitch` | `ankle_pitch` | broadcast identical. |

The broadcast leg-pitch is **not** the planted-feet solution (broadcasting it directly makes the feet
fly up; see §6). It is used only as a **low-weight posture seed / initial guess** to bias the OCP into
the correct **knee-flexion branch** (the unconstrained solve otherwise finds a hyperextended branch).

### Joints — SOLVED by the OCP (free; weak nominal nullspace regularization)
`L/R Hip_Roll`, `L/R Hip_Yaw`, `L/R Ankle_Roll`, `Head`×2. The legs are determined by the hard
planted-feet contact + the base-pose reference + joint limits. **Do not hard-hold the redundant leg
DOF at nominal** (that fights contact); weak regularization only, for nullspace conditioning.

### Hands — task-space targets
`left_hand_xyz/quat`, `right_hand_xyz/quat` → `anchor_pose(xyz, quat, x0, y0, yaw0)`: XY rotated by
`R_z(yaw0)` then `+(x0,y0)`, **Z passthrough**, quat pre-composed with `R_z(yaw0)`. Tracked at
`left_hand_link/right_hand_link`. **Position** is tracked (soft, high weight) + **hard at grasp
keyframes** (§5, §7). Hand **orientation** is not separately constrained (it follows from the tracked
arm joints). Hand velocity `/time_scale`.

### Velocities
Reference velocities for tracked channels are finite-differenced at native segment `dt` at load, then
scaled `/time_scale` at sample time. Base lin/ang velocities rotated to body-local per the §9
state-map convention. Untracked-joint `v_ref = 0`.

## 5. Grasp keyframes (auto-detected from `held_objs`)

held_objs transitions (segment boundaries where a hand's hold state flips) give 4 events on the global
timeline (each segment = 81 samples): **left-grasp @ sample 81**, **right-grasp @ 243**,
**left-release @ 405**, **right-release @ 567**. At each, the relevant hand gets a **hard hand-position
constraint** over a small node window (±W nodes, W≈1–2 tunable) via the slack trick (§7). A node within
a grasp window for foot/hand `h` sets grasp gate `g_h = 1`, else `0`.

## 6. Leg handling — validation (why "solve the legs", not "broadcast")

Empirically validated (scratch probes, 2026-06-28):
- Broadcasting plan leg-pitch onto hips/knees/ankles with the Trunk as floating base drives the feet
  **+46 to +83 cm off the ground** — geometrically inconsistent with planted feet.
- A leg-IK over the **full motion** that holds the base at the plan pose (z=`trunk_height`, lean=quat)
  and pins both feet flat on the ground converges with **max foot error < 0.03 mm** → the base pose is
  reachable with planted flat feet throughout.
- Unconstrained, that IK finds a **hyperextended-knee branch** (knee < 0, out of `[0,2.34]`). With
  **joint-position bounds + a knee-flexion seed**, it converges in-limit over the whole motion (max foot
  error < 0.02 mm, 0/72 frames off).

**Conclusion:** track the base pose + arms + hands; let the OCP solve the legs against hard planted-feet
contact, **with leg joint-position limits** (prevent hyperextension) and the **leg-pitch seed** (select
the flexion branch). No leg IK in the controller.

## 7. The OCP — `PickupOCP`

Full-order whole-body RNEA, all-stance (8 corners always in contact), Fatrop interior-point, compiled
via `opti.to_function` (NO JIT — the expanded graph is too large to gcc-compile in acceptable time).

### Horizon (real-time sized)
- **N = 10 nodes (target), N = 8 (fallback)**, uniform **`dt = 0.04`** default (horizon 0.40 s, matching
  `t1_kd_mpc`). N and dt are config-tunable. Rationale & data: §8.
- `tau_nodes = 3` (torque variables on the first 3 nodes only), unchanged.

### Decision variables (interleaved `DX[0],U[0],…,DX[N]` — Fatrop staircase)
- `DX[i] ∈ ℝ⁷⁰` (`i=0..N`); absolute state `x_i = integrate(x_init, DX[i])`.
- `U[i] = [ a(35), forces(24), s_hand(6), tau_j(29 iff i<tau_nodes) ]`
  - `a` generalized accelerations, `forces` 8 corner 3D forces (world),
  - `s_hand` = 2 hands × 3 = 6 hand task-space slacks (the slack trick),
  - `tau_j` joint torques (first `tau_nodes` nodes only).

### Parameters (set per tick)
`x_init(71)`, `Q_diag(70)`, `R_diag(.)`, **`x_ref(71, N+1)`** (per-node reference state),
**`hand_ref(6, N+1)`** (L,R hand world positions), **`grasp_gate(2, N+1) ∈ {0,1}`**,
**`leg_seed(6, N+1)`** (the 6 broadcast leg-pitch values: L/R hip_pitch, knee, ankle_pitch — for
regularization). The `(x0,y0,yaw0)` anchor is baked into the refs at sample time in `reference.py`.

### Constraints per node (ordering: gap-closing FIRST, then equalities, then inequalities)
1. **Gap-closing (forward Euler):** `dq_{i+1} == dq_i + v·dt`, `dv_{i+1} == dv_i + a·dt`. (Slow,
   quasi-static motion → Euler is adequate; the midpoint integrator from the walk work is not needed.)
2. **Floating-base underactuation:** `rnea(q,v,a,forces)[:6] == 0` (all 8 corners active).
3. **Torque** (`i<tau_nodes`): `rnea(...)[6:] == tau_j`, `-tau_max ≤ tau_j ≤ tau_max`.
4. **Contact** (`i≥1`), both feet planted: `V_lin == 0` (foot-center linear velocity), `ω_x=ω_y=0`
   (flat foot). All 8 corner friction cones (`fz≥0`, `μ²fz² ≥ fx²+fy²`, `μ=0.4`).
5. **Hand task (slack trick, `i≥1`):** for each hand `h`, `hand_pos_h(q) − hand_ref_h == (1−g_h)·s_h`.
   - `g_h=0` (non-grasp): `hand_pos − ref = s_h` free, with soft cost `w_hand·‖s_h‖²` → soft tracking.
   - `g_h=1` (grasp): `hand_pos == ref` hard. Full Jacobian both modes (no degeneracy).
6. **Leg joint position limits (NEW):** `q_lo ≤ q_leg_i ≤ q_hi` on the 12 leg joints (≥ a few nodes;
   at minimum prevents hip/knee/ankle hyperextension). Implemented as box inequalities on `q_i` leg
   components. (Optionally extend to all joints if cheap.)

### Objective
- State tracking: `Σ_i (DX_i − dx_ref_i)ᵀ Q (DX_i − dx_ref_i)` + terminal, where `dx_ref_i =
  difference(x_init, x_ref_i)`. `Q` weights: high on base z + orientation (lean) + arm joints + waist;
  **low** on base x/y/yaw, **low** on leg joints (seed weight only), velocities modest.
- Hand task: `Σ_i w_hand · ‖s_h‖²` (high weight; hands are top-priority à la `t1_kd_mpc`'s 400).
- Leg-pitch seed: `Σ_i w_leg · ‖q_legpitch_i − leg_seed_i‖²`, `w_leg` small (branch bias only).
- Input regularization `R`: accel→0, forces→gravity-over-corners, slacks→0, torque→0.

### Solver path & warm start
- `opti.to_function("pickup_fn", [x_init, Q, R, x_ref, hand_ref, grasp_gate, leg_seed, opti.x(, opti.lam_g)], [opti.x(, opti.lam_g)])`.
- Warm start: pass previous solution as `opti.x` (primal). Carrying `opti.lam_g` (dual) as I/O is an
  option if it measurably reduces warm iterations; evaluate during tuning.
- Fatrop opts as M0 (`structure_detection='auto'`, `expand=True`, `print_level=0`, `tol=1e-3`); warm
  ticks stop early well under `max_iter`.

## 8. Real-time strategy

- **Lever = horizon reduction.** JIT/codegen is ruled out (the expanded full-order RNEA graph is too
  large to gcc-compile in acceptable time — confirmed 2026-06-28).
- Measured deployment-path (`to_function`, no JIT, warm) **stand** sweep @ 50 Hz / 20 ms budget:

  | N | horizon | warm p50 | warm p90 |
  |---|---|---|---|
  | 31 | 1.09 s | 43.9 | 50.8 |
  | 20 | 0.70 s | 30.4 | 33.4 |
  | 16 | 0.56 s | 21.6 | 22.3 |
  | 12 | 0.42 s | 18.9 | 20.3 |
  | **10** | **0.35 s** | **14.2** | **16.3** |
  | 8 | 0.28 s | 13.7 | 14.9 |

  (Machine/load-dependent; numbers are internal-relative. M0 doc reports stand N=31 ≈ 28.7 ms p90 on a
  quieter machine.)
- `PickupOCP` adds hand-tracking + slack + leg-limit cost over bare stand, so it runs somewhat slower at
  equal N. **Target N=10; fall back to N=8** if measured pickup p90 ≥ 16 ms. N is config-tunable; the
  acceptance gate is *measured* p90 < 16 ms.
- `time_scale` is a second real-time lever: slowing the motion makes tick-to-tick reference change tiny
  → near-perfect warm starts → fewer iterations, and permits a lower MPC rate if ever needed (the
  500 Hz PD handles fast stabilization).
- Rates: MPC 50 Hz, control (PD) 500 Hz, physics 2000 Hz (unchanged from stand).
- **Fallback ladder** (apply in order only if budget missed): N 10→8→6 → tighten warm `max_iter` →
  8→4 contact corners (per-foot wrench; changes contact model, log divergence) → lower MPC rate
  (leverage `time_scale`). Model reduction (centroidal/KD) is explicitly out of scope.

## 9. time_scale & reference sampling

- **Convention: larger = slower** (matches `t1_kd_mpc`, default there 5.0). `t_ref = clip(t_wall /
  time_scale, 0, T_end)`. Positions/quaternions are unscaled (same path, traversed slower); all
  velocities `/time_scale`. Default `time_scale` exposed as a CLI/config arg (start ~3–5).
- 8 segments concatenated into one global timeline (cumulative per-segment `dt`, seam-dedup).
- **Sampling onto the OCP grid with interpolation** (improvement over `t1_kd_mpc`'s nearest-frame):
  linear for joints/positions, **slerp** for base & hand quaternions, at each node phase
  `t_ref + k·dt_node`. Beyond `T_end`, clamp to the final frame (hold).
- MuJoCo↔pinocchio base-velocity rule (the §9 map in `wb/state.py`) is the single source of truth for
  world↔body-local velocity conversion; do not re-derive.

## 10. Components / file structure

- `t1_nmpc/wb/reference.py` (NEW) — `MotionPlanReference`:
  - `load(path)`; build global timeline; `frame_to_xref(seg,k) → x_ref71` (the §4 mapping);
    grasp-keyframe detection (§5); `sample(t_wall, time_scale, node_times) → (x_ref[N+1], hand_ref[N+1],
    grasp_gate[N+1], leg_seed[N+1])` with interpolation/slerp.
  - Pure NumPy/pinocchio; no CasADi. Single source of truth for the mapping.
- `t1_nmpc/wb/track_ocp.py` (NEW) — `PickupOCP`: the §7 OCP (build once, `to_function`).
- `t1_nmpc/wb/track_mpc.py` (NEW) — `TrackingMPC`: holds plan + OCP; per tick advances the phase clock,
  samples the reference over the horizon, calls the solver fn (warm), extracts the node-1 `JointCommand`
  (reuse `wb/state.first_node_command`/`extract_command`).
- `sim/pickup.py` (NEW) — closed-loop MuJoCo runner mirroring `sim/stand.py`; args
  `--time_scale --duration --view --gif [--realtime]`. `--realtime` paces to wall clock and reports the
  real-time factor + missed deadlines. Metrics in §1.
- Config: extend `MPCConfig` (or a `make_track_config`) with `nodes` default for tracking (10),
  `time_scale`, hand/leg weights, grasp window `W`. Keep M0 `make_config()` defaults intact.
- Tests under `tests/` (§12).

## 11. Deliberate divergences (log in `docs/2026-06-25-t1controller-divergences.md`)

1. **Hard hand-position constraint at grasp keyframes** — `t1_kd_mpc` keeps hands always-soft
   (top weight 400). We harden at the isolated grasp instants for grasp accuracy; risk: conflict with
   hard planted-feet + RNEA → infeasibility. Mitigation: verify Fatrop feasibility at grasp nodes;
   auto-fall back to high-weight soft if a grasp node is infeasible.
2. **Base height tracked from `trunk_height`** — `t1_kd_mpc` ignores plan base position; we use it
   (validated: §6).
3. **Interpolation (linear + slerp)** vs `t1_kd_mpc` nearest-frame sampling.
4. **Leg joint-position limits** added to the OCP (M0 stand has only torque limits).
5. **No payload model** — same as `t1_kd_mpc` (object mass not in the plan); flagged as future work.

## 12. Test plan

- `test_reference_mapping` — FK round-trip: plan frame → `x_ref` → `left/right_hand_link` FK matches
  plan `hand_xyz` to < 1 mm; `Waist == −trunk_yaw`; arm order correct; base z == `trunk_height`.
- `test_reference_timescale` — velocities scale `1/time_scale`; positions unchanged; phase clock maps
  `t_wall/time_scale`; total wall duration = `T_ref·time_scale`.
- `test_grasp_keyframes` — detected events at samples 81/243/405/567 with correct hand.
- `test_pickup_ocp_solves` — cold + warm solve at default N; `success=True`; feet stay planted in the
  solution; hand error small at a grasp gate.
- `test_pickup_realtime` — warm solve p90 < 16 ms at the chosen N (skip/xfail-marked if machine slow,
  but record the number).
- `test_pickup_closed_loop` — short `sim/pickup.py` run: completes, no fall, hand-at-grasp < 2 cm,
  Σfz/mg ≈ 1.
- M0 regression: existing `tests/` stand cases still pass.

## 13. Non-goals (v1)

Payload/object mass-inertia, hand-orientation hard tracking, walking/stepping, hardware/SDK, JIT or
C-codegen of the solver, multi-plan generalization.

## 14. Open risks

- **Pickup OCP at N=10 may exceed 16 ms** once hand/slack/leg-limit costs are added → fall to N=8 (ladder
  §8). Decide by measurement early (build the OCP, benchmark, before wiring the closed loop).
- **Grasp hard-constraint feasibility** under hard contact + RNEA → soft fallback (§11.1).
- **Deep-crouch balance** (CoM far forward when reaching the floor): double-support polygon is large and
  the plan is pre-validated, but confirm Σfz/mg and CoP stay sane; raise base-orientation / add a CoM
  term if needed (tuning, not architecture).
